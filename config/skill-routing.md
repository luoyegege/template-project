# 技能路由规则

> 涉及技能调用时，先查本文档确认分组和冲突规则，再执行。同组内有多个候选时，按「何时不用」排除，剩下的就是正确选择。不确定时，列出候选技能和理由，让爷决定。
> 中文触发词：补充英文触发条件，不替代。
>
> **本表只覆盖设备级共享池技能**（任何项目 clone 后即有）。本项目专属技能创建后追加到对应分组。

## 功能分组

### 图片生成组
| 技能 | 用途 | 何时不用 |
|------|------|----------|
| `image2` | DreamMaker gpt-image-2 单图生成 | 需要设计排版时不用（用 canvas-design） |
| `canvas-design` | 生成有设计排版的海报/宣传图/可视化图表 | 纯 AI 生图无需排版时不用 |
| `banner-studio` | 运营图工坊：素材库 + 图层编排出图 | 单张 AI 图足够时不用 |

**消歧规则：** 单张 AI 图 → `image2`；设计排版（海报/图表，代码画布非 AI） → `canvas-design`；多素材编排出图 → `banner-studio`。

## 技能选择行为规则

1. **先查分组**：涉及技能调用时，先找到对应功能组，再看「何时不用」排除候选。
2. **冲突时看消歧规则**：同组有多个候选，按消歧规则选，不猜测。
3. **不确定时说出来**：列出候选技能和选择理由，让爷决定，不默默猜。
4. **新技能创建后必须更新本文档**：skill-creator 完成技能后，将新技能加入对应分组（或新建分组）。
5. **创建技能前先确认目录**：项目专属技能写入 `<项目根>/skills/<技能名>/`；通用技能应推到共享池 `~/ClaudeAI/shared-skills/skills/`，不要写到全局 `~/.claude/skills/`（那是联合出来的，会被重建覆盖）。
6. **官方插件（`/plugin`）管理**：插件装在 `~/.claude/plugins/`，由 `/plugin` 命令独立管理，和本项目 `skills/`、共享池物理隔离。装插件前必查同名：插件/其内含 skill 与共享池同名且同上游源 → 跳过；同名不同源 → 报告爷决策。
